UCL QuizApp – User guide

Guide content

The following guide explains how to install and use the UCL QuizApp, a location-based mobile app. Here is the content of the guide:

-        System requirements;

-        How to install the app;

-        How the app works (functionalities of both the left-side and the top-right menu).

System requirements

The app works on Android OS, while iOS and Windows OS are not supported. The minimum supported version of Android is 5.1.0 (further information about the Android versions supported on the Phonegap website).

The app must to be connected to the UCL VPN to load the points of interest. We recommend the Cisco AnyConnect app to establish the VPN connection; further information about how to connect to the UCL VPN can be found on the following link.

How to install the app

Since the app is not distributed through the Google Play Store, the user must allow the installation from unknown sources on his device, otherwise the app cannot be installed. Usually, this option is under the security section within the settings.

The installation process needs to be initialized from the following link: https://build.phonegap.com/apps/3148274/share.

Once here the user has two options:

1-     Through their own mobile phone by scanning the QR code provided and by following the instructions supplied by the device to complete the process;

2-     By downloading the .apk file on the PC and then connecting the phone to the computer to complete the installation on the mobile device.

How the app works

This is a location-based app that uses the capability of mobile devices to track the position of the user in order to visualise questions related to some points of interest (mainly related to the UCL campus). The app is composed of three main parts: the main body, the left-side menu and the top-right menu.

The main body

The main body contains the visualisation tools that allow the user to get the questions and answer to the quiz. This part consists of:

-        The map: here the user can see their position and the location of the points of interest. Each point is associated to a question to be answered.

-        The coordinates of the tracked position: this option lets the user see the coordinates of their position expressed in decimal degrees (further information about the coordinates can be found here).

-        The question: this section contains the question and the related answers that appear when the user is near enough to a point of interest.

-        The check answer button: once the user selects an answer they can check if it is correct by clicking this button.

The left-side menu

This menu can be opened by tapping the top-left burger on the interface. Here the user has four options:

-        Load Points: the button allows to load the points of interest. Before starting the game the points need to be loaded, or else there are no questions to answer.

-        Track Location: once clicked, the option activates the tracking and search for points nearby. If the user is close to one of them a pop-up will appear and the question will be loaded, otherwise another pop-up will notify the user that they are far from any point of interest.

-        Get Questions: the user can check if there is a point near enough by clicking this option. If so, a pop-up will appear and the question will be loaded; if not another pop-up will notify the user that they are far from any point of interest.

-        Clear View: this option cleans the map. The user should use this option if they want to restart the game.

The top-right menu

From the top-right menu the user can access both this user guide (User Guide link) and additional documentation that gives more information about the app and its functionalities (More Info link).
